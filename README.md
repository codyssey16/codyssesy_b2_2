# 📰 [Project B] 뉴스 요약 자동화 워크플로우

## 📌 1. 프로젝트 개요

자동화 플랫폼을 활용하여 외부 데이터 소스(RSS 피드)로부터 최신 기술 뉴스를 자동으로 수집하고, 생성형 AI를 통해 내용을 핵심 요약하여 노션(Notion) 데이터베이스에 자동 저장하는 노코드(No-Code) 기반의 자동화 파이프라인 프로젝트이다.

* **프로젝트 기간**: 2026.06.22(월) ~ 2026.06.28(일) (40시간)

## 🛠️ 2. 활용 도구 및 주요 기능

* **자동화 플랫폼**: `Relay.app`
* **AI 모델**: `GPT 5.4 mini`(요약) / `GPT 5.4 nano`(감정 분석)
* **데이터베이스**: `Notion`, `Google Sheet`
* **RSS 주소**: [TechCrunch 기술 뉴스 RSS 피드](https://techcrunch.com/feed/)

* **주요 기능**:

  - [x] **데이터 수집**: 매일 정해진 시간에 스케줄러가 트리거되어 RSS 피드로부터 기술 뉴스를 자동 수집.
  - [x] **주제 필터링**: 수집된 뉴스 중 설정한 주제(AI 등)에 맞는 최신 기사 1건을 선별하여 본문 및 메타데이터를 추출.
  - [x] **AI 가공**: 생성형 AI(OpenAI)를 활용하여 기사 본문을 3줄 이내의 간결한 문장으로 자동 요약.
  - [x] **자동 저장**: 노션 데이터베이스의 속성(Title, Summary, URL, Date, Emotion, Guid)에 가공된 데이터를 저장.
  - [x] **안정성 및 비용 최적화**: 중복 방지 키 적용, API 호출 1회 원칙 및 최대 2회 재시도 제한을 준수하여 설계.
  - [x] **보너스 - 감성 분석 태그 추가**: 뉴스 본문의 긍정/부정 감성을 AI로 분석하여 태그로 분류.

## ⚠️ 5. 에러 처리 정책 및 선택 이유 (이 부분은 지속적 확인 필요)

* **정책 설정**:
  1. **중복 저장 방지**: Rss 내 Guid tag 를 기준값으로 설정하여 데이터베이스 내 중복을 차단
  2. **API 호출 제한**: 요약용 AI 호출은 기사 1건당 **최대 1회**를 원칙으로 하여 불필요한 비용 발생을 방지.
  3. **재시도(Retry) 메커니즘**: 네트워크 불안정 등 워크플로우 실행 불가로 인한 일시적 오류 발생 시, **최대 2회**까지 자동 재시도를 수행. 
    **최종 실패 처리**: 2회 재시도 후에도 실패할 경우, 전체 워크플로우가 멈추지 않도록 해당 건을 `Skip(로그 기록)`으로 처리하고, 구글 시트에 **에러** 행 추가.
* **선택 이유**: 무한 루프로 인한 API 비용 과다 청구를 방지하고, 소수 기사의 오류가 전체 자동화 파이프라인의 중단으로 이어지지 않도록 시스템 안정성을 확보하기 위함.
