# 📰 [Project B] 뉴스 요약 자동화 워크플로우

## 📌 1. 프로젝트 개요

자동화 플랫폼을 활용하여 외부 데이터 소스(RSS 피드)로부터 최신 기술 뉴스를 자동으로 수집하고, 생성형 AI를 통해 내용을 핵심 요약하여 노션(Notion) 데이터베이스에 자동 저장하는 노코드(No-Code) 기반의 자동화 파이프라인 프로젝트이다.

* **프로젝트 기간**: 2026.06.22(월) ~ 2026.06.28(일) (40시간)

## 🛠️ 2. 활용 도구 및 주요 기능  

* **자동화 플랫폼**: `Make`
* **AI 모델**: `GPT-4o-mini`(기사 1건 선택) / `GPT-4.1-mini`(기사 요약 및 감정 분석)
* **데이터베이스**: `Notion`, `Google Sheet`
* **RSS 주소**: [TechCrunch 기술 뉴스 RSS 피드](https://techcrunch.com/feed/)

* **주요 기능**:

  - [x] **데이터 수집**: 매일 정해진 시간에 스케줄러가 트리거되어 RSS 피드로부터 기술 뉴스를 자동 수집.
  - [x] **주제 필터링**: 수집된 뉴스 중 설정한 주제(AI 등)에 맞는 최신 기사 1건을 선별하여 본문 및 메타데이터를 추출.
  - [x] **AI 가공**: 생성형 AI(OpenAI)를 활용하여 기사 본문을 3줄 이내의 간결한 문장으로 자동 요약.
  - [x] **자동 저장**: 노션 데이터베이스의 속성(Title, Summary, URL, Date, Emotion, Guid)에 가공된 데이터를 저장.
  - [x] **안정성 및 비용 최적화**: 중복 방지 키 적용, API 호출 1회 원칙 및 최대 2회 재시도 제한을 준수하여 설계.
  - [x] **보너스 - 감성 분석 태그 추가**: 뉴스 본문의 긍정/부정 감성을 AI로 분석하여 태그로 분류.

## ⚠️ 5. 에러 처리 정책 및 선택 이유

본 워크플로우에는 시스템 에러(System Error)와 데이터 정제 오류(Business Logic Issue)를 다각도로 방어하는 2중 안전망이 설계되었다. 아래는 요약본이므로 더 자세한 사항은 [[여기]](document/result_error_details.md)를 참조하길 권한다.

### 1) 시스템 에러 대응 정책 (Error Handling)

에러의 치명도(Criticality)에 따라 즉각 조치(이메일+재시도)와 흐름 유지(구글 시트 기록+스킵) 정책을 분리 적용하였다.

| 대상 모듈 | 에러 상황 (테스트 케이스) | 대응 정책 | 선택 이유 (실무적 근거) |
| :--- | :--- | :--- | :--- |
| **RSS / Notion / OpenAI** | 피드 주소 오류, API 할당량 초과, 세션 만료, 네트워크 불안정 등 | **관리자 이메일 알림 + Retry(재시도)** | 핵심 파이프라인의 장애는 전체 시스템 중단을 의미하므로, 즉시 관리자에게 알리고 시간 차 재시도를 통해 데이터 유실을 최소화. |
| **HTTP / Parse JSON** | 타겟 사이트 다운, OpenAI의 비정상 JSON 반환 등 | **Google Sheets 로그 기록 + Skip** | 특정 기사의 스크래핑/파싱 실패는 부차적인 문제이므로, 해당 데이터만 구글 시트에 기록 후 시스템이 멈추지 않고 다음 기사를 처리하도록 보장함. |

### 2) 비즈니스 로직 및 예외 처리 정책 (Routing & Filtering)

데이터의 정합성을 유지하고 불필요한 AI 토큰 낭비 및 데이터베이스 오염을 방지하기 위한 필터링 체계이다.

* **기사 중복 방지 (GUID 체크):** 매일 스케줄러가 작동할 때 기존 노션 DB에 존재하는 고유 식별자(`Guid`)를 가진 기사는 필터링하여 중복 적재를 방지하고 저장 공간을 효율적으로 관리.
* **AI 뉴스 키워드 필터링:** 수집된 기사 중 핵심 키워드가 없는 경우 Router 우회 경로를 통해 금일은 AI기사가 없음을 Google Sheets(issue_log)에 기록.
* **HTTP 응답 코드 검증 (Status Code 200~299):** 정상 페이지가 아닌 에러 페이지(404, 500 등)의 텍스트가 파서(Text Parser)로 넘어가 토큰을 낭비하거나 잘못된 요약을 생성하는 것을 차단.
* **JSON 필수 데이터 검증:** 문법은 정상이나 필수 데이터(예: `Summary`)가 누락된 불완전한 데이터는 Router의 Fallback 경로를 통해 구글 시트에 기록하여 노션 DB의 데이터 정제성을 유지.
* **API 호출 제한**: 1차 `OpenAI`가 중요한 기사 1건을 수집된 AI 기사들의 Title과 Description으로만 선택한 후 URL를 출력하면 이 주소만 `HTTP`와 `Text Parser`를 거치고 그 결과물(기사 1건의 본문)을 2차 `OpenAI`가 요약하여 요약 호출은 기사 1건당 최대 1회를 원칙을 지키고 불필요한 비용 발생을 방지.
